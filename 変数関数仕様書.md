# 変数関数仕様書

## 1. モジュール: `config`
設定管理を司るモジュール。環境変数、TOMLファイル、デフォルト値を統合して管理する。

### 1.1. 構造体: `Config`
アプリケーション全体のメイン設定構造体。

| フィールド | 型 | 説明 |
| :--- | :--- | :--- |
| `discord` | `DiscordConfig` | Discord通知に関する設定 |
| `general` | `GeneralConfig` | 全般的な通知設定 |

- **依存関係 (Dependencies)**:
    - `DiscordConfig`: Discord固有の設定を保持。
    - `GeneralConfig`: アプリ全般の設定を保持。
    - `serde::Serialize`, `serde::Deserialize`: 設定のシリアライズ/デシリアライズに使用。
- **影響範囲 (Impact Scope)**:
    - `main.rs`: アプリケーション起動時の設定保持に使用。
    - 今後の通知処理モジュール: 通知先URLやタイトルの取得元となる。

### 1.2. 構造体: `DiscordConfig`
Discord Webhookに関する設定。

| フィールド | 型 | 説明 |
| :--- | :--- | :--- |
| `webhook_url` | `Option<String>` | Discord WebhookのURL。環境変数 `DISCORD_WEBHOOK_URL` で上書き可能。 |

- **依存関係 (Dependencies)**: なし
- **影響範囲 (Impact Scope)**: `Config` 構造体の一部として利用。Discordへのリクエスト送信時にURLとして使用される。
- **セキュリティ (Security)**: `std::fmt::Debug` を手動で実装し、デバッグログ出力時に `webhook_url` の中身を `"********"` にマスクする処理を行っている。

### 1.3. 構造体: `GeneralConfig`
全般的な動作設定。

| フィールド | 型 | 説明 |
| :--- | :--- | :--- |
| `default_title` | `String` | 通知のデフォルトタイトル。環境変数 `OS_NOTIFICATION_DEFAULT_TITLE` で上書き可能。 |

- **依存関係 (Dependencies)**: なし
- **影響範囲 (Impact Scope)**: `Config` 構造体の一部として利用。通知メッセージのデフォルトタイトルとして全通知処理に影響する。

### 1.4. 関数: `Config::load()`
設定を読み込む。

- **引数**: なし
- **戻り値**: `Config`
- **処理内容**:
    1. デフォルト値で初期化。
    2. ユーザーの設定ディレクトリ（Windowsなら `%AppData%\mcp-notifier`）から `config.toml` を探す。
    3. ファイルが存在しなければデフォルト値を書き込み、存在すれば読み込む。
    4. 環境変数が設定されていれば、その値で最終的な上書きを行う。
- **依存関係 (Dependencies)**:
    - `directories-next`: OSごとの適切な設定ディレクトリ取得に使用。
    - `toml`: 設定ファイルのパースと書き出しに使用。
    - `std::env`: 環境変数の取得に使用。
    - `std::fs`: 設定ファイルの読み書きに使用。
- **影響範囲 (Impact Scope)**:
    - `main.rs`: 起動時に呼ばれ、アプリ全体の挙動を決定する。
    - ファイルシステム: `%AppData%` 内に設定ファイルを生成・更新する副作用がある。
    - 標準エラー出力: デバッグログやエラー情報を `eprintln!` で出力する。

## 2. モジュール: `main`
アプリケーションのエントリポイント。CLI引数の解析とモード切替を行う。

### 2.1. 構造体: `Cli`
コマンドライン引数を定義するメイン構造体。

| フィールド | 型 | 説明 |
| :--- | :--- | :--- |
| `mcp` | `bool` | `--mcp` フラグ。真の場合、MCPサーバーモードで動作する。 |
| `command` | `Option<Commands>` | サブコマンド（`os` または `discord`）。 |

- **依存関係 (Dependencies)**:
    - `clap::Parser`: コマンドライン引数のパースに使用。
- **影響範囲 (Impact Scope)**:
    - アプリケーション全体の動作モード（MCP/CLI）を決定する。
    - 各サブコマンドに対応する `notifier` モジュールの関数を呼び出す。

### 2.2. 列挙型: `Commands`
CLIモードでの動作を定義するサブコマンド。

| バリアント | 引数 | 説明 |
| :--- | :--- | :--- |
| `Os` | `title: Option<String>`, `message: String` | OS通知を送信する。 |
| `Discord` | `content: String`, `prompt: Option<String>` | Discord Webhook通知を送信する。 |

- **依存関係 (Dependencies)**: なし
- **影響範囲 (Impact Scope)**:
    - 各通知処理の実行。 `notifier` モジュールの対応する関数に引数を渡して呼び出す。

## 3. モジュール: `notifier`
通知の送信処理を担うモジュール。

### 3.1. 関数: `send_os_notification(title: &str, message: &str)`
OSのデスクトップ通知を送信する。

- **引数**:
    - `title`: 通知のタイトル。
    - `message`: 通知の本文。
- **処理内容**: `notify-rust` を使用して、アプリケーション名「Oshirase-MCP」でデスクトップ通知を表示する。タイムアウトは設定せず、ユーザーが閉じるまで表示し続ける。
- **依存関係 (Dependencies)**:
    - `notify-rust`: OS固有の通知API呼び出しに使用。
- **影響範囲 (Impact Scope)**:
    - ユーザーのデスクトップ環境。
    - `main.rs`: `os` サブコマンド実行時に呼び出される。
    - 今後のMCPプロトコル実装: 通知アクション実行時に呼び出される。

### 3.2. 関数: `send_discord_notification(webhook_url: &str, content: &str, prompt: Option<&str>, default_title: &str)`
Discord Webhookを使用してリッチな通知を送信する。

- **引数**:
    - `webhook_url`: Discord WebhookのURL。
    - `content`: 通知のメインコンテンツ（Embedのdescriptionに使用）。
    - `prompt`: 任意。送信のきっかけとなったプロンプト等。
    - `default_title`: Embedのタイトル。
- **処理内容**:
    1. `reqwest` クライアントを作成。
    2. `prompt` が存在する場合、冒頭100文字を Embed のフィールドとして構成。
    3. ネオンカラー（0x00ff00）の Embed を含む JSON ペイロードを作成。
    4. 指定された Webhook URL に対して POST リクエストを非同期で送信。
- **依存関係 (Dependencies)**:
    - `reqwest`: HTTPリクエスト送信に使用。
    - `serde_json`: ペイロードのJSON構築に使用。
    - `chrono`: タイムスタンプの取得に使用。
- **影響範囲 (Impact Scope)**:
    - 指定されたDiscordチャンネル。
    - `main.rs`: `discord` サブコマンド実行時に呼び出される。
    - 今後のMCPプロトコル実装: 通知アクション実行時に呼び出される。

### 2.3. 構造体: `OshiraseHandler`
MCP (Model Context Protocol) のリクエストを処理するハンドラ。

| フィールド | 型 | 説明 |
| :--- | :--- | :--- |
| `config` | `Arc<Config>` | スレッドセーフに共有される設定情報。 |

- **依存関係 (Dependencies)**:
    - `mcp_sdk_rs::server::ServerHandler`: MCPハンドラのトレイトを実装。
    - `Arc`: 複数スレッド間での設定共有に使用。
- **影響範囲 (Impact Scope)**:
    - MCPサーバーとしての中心的な挙動（ツールのリストアップ、呼び出し）を制御する。
    - `notifier` モジュールの各関数を呼び出し、実際の通知処理を実行する。

### 2.4. トレイト実装: `impl ServerHandler for OshiraseHandler`
MCPサーバーの各メソッドを実装する。

- **実装メソッド**:
    - `initialize`: サーバーの機能をクライアントに通知する。`tools` を有効に設定。
    - `shutdown`: サーバーの終了処理。
    - `handle_method`: `tools/list` および `tools/call` を処理する。
        - `tools/list`: `notify_os` と `notify_discord` の定義を返す。
        - `tools/call`: 指定されたツールを引数とともに実行し、結果を `CallToolResult` として返す。
- **依存関係 (Dependencies)**:
    - `serde_json`: MCPのリクエスト/レスポンス形式（JSON-RPC）のパースと生成に使用。
    - `notifier`: 各ツールの実体としての関数呼び出し。
- **影響範囲 (Impact Scope)**:
    - AIエージェント（Gemini CLI等）との連携インターフェース。
    - 標準エラー出力: ツールの呼び出し履歴や警告を `eprintln!` で出力する。
